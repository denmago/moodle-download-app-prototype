A download page for the Moodle app. Needed to feel real enough for user testing — with animations, a QR code, and a mobile bridge page.

The workflow

Figma MCP → Claude Code → Browser

1
Share a Figma URL
Claude Code connects directly to Figma via MCP — no exporting, no screenshots. It reads exact tokens, assets, and layout specs.
2
Describe what you want
Plain language prompts. "Make the corners round inside the phone." "Add an entrance animation." "Generate a real QR code."
3
Iterate in real time
Every change lands in the browser instantly. No build step, no staging environment. Just save and refresh.

What I learnt

Designers can ship prototypes
without a developer

Figma MCP is a superpower
Claude reads exact tokens and assets from your file. No more "can you export this PNG" conversations.
Specificity beats vagueness
The more precise the prompt ("32px circle, colour F3CEE5, icon 32×32px"), the better the output.
Iteration is the process
Don't expect perfection on the first prompt. Treat it like a design critique — give feedback, refine, repeat.
Try it yourself
